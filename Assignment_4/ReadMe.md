**Ответы на контрольные вопрсоы:**

**1.В чём заключается отличие гибридных вычислений от вычислений только на CPU или только на GPU?**

Отличие гибридных вычислений от вычислений только на CPU или только на GPU заключается в том, что гибридный подход использует сильные стороны обоих типов устройств одновременно. CPU хорошо справляется с управлением, логикой, ветвлениями и нерегулярными задачами, тогда как GPU эффективен для массово-параллельных вычислений над большими объёмами данных. В вычислениях только на CPU параллелизм ограничен числом ядер, а при использовании только GPU возникают издержки на передачу данных и сложности с алгоритмами, плохо поддающимися параллелизации. Гибридные вычисления позволяют распределить работу более рационально и повысить общую производительность.

**2.Для каких типов задач целесообразно распределять вычисления между CPU и GPU?**

Распределять вычисления между CPU и GPU целесообразно для задач, где есть чёткое разделение на управляющую часть и вычислительно интенсивные фрагменты. Например, CPU может подготавливать данные, управлять потоками и принимать решения, а GPU выполнять матричные операции, обработку изображений, численные расчёты или симуляции. Такой подход особенно эффективен, когда объём вычислений на GPU значительно превышает накладные расходы на передачу данных.

**3.В чём разница между синхронной и асинхронной передачей данных между CPU и GPU?**

Разница между синхронной и асинхронной передачей данных между CPU и GPU заключается в поведении программы во время копирования данных. При синхронной передаче выполнение программы блокируется до полного завершения копирования, из-за чего вычислительные ресурсы могут простаивать. При асинхронной передаче копирование происходит параллельно с вычислениями, и программа может продолжать выполнение других операций, не дожидаясь окончания передачи данных.

**4.Почему асинхронная передача данных может повысить производительность программы?**

Асинхронная передача данных повышает производительность, потому что позволяет перекрывать вычисления и обмен данными. Пока GPU обрабатывает один набор данных, CPU может подготавливать следующий или выполнять другие вычисления. Это уменьшает время простоя устройств и улучшает загрузку ресурсов, особенно в задачах с частыми операциями копирования.

**5.Какие основные функции MPI используются для распределения и сбора данных между процессами?**

Основные функции MPI для распределения и сбора данных включают механизмы отправки и приёма сообщений, а также коллективные операции. На практике часто используются функции для точечной передачи данных между процессами и коллективные вызовы, такие как рассылка данных всем процессам, сбор результатов и их агрегирование в одном процессе, что упрощает реализацию распределённых алгоритмов.

**6.Как количество процессов MPI влияет на время выполнения программы и почему?**

Количество процессов MPI влияет на время выполнения программы двояко. С увеличением числа процессов вычислительная нагрузка распределяется, и время выполнения может уменьшаться. Однако после определённого момента начинают доминировать накладные расходы на коммуникации, синхронизацию и управление процессами, из-за чего дальнейшее увеличение числа процессов либо не даёт выигрыша, либо даже замедляет выполнение.

**7.Какие факторы ограничивают масштабируемость распределённых параллельных программ?**

Масштабируемость распределённых параллельных программ ограничивается задержками передачи данных по сети, пропускной способностью коммуникационных каналов, дисбалансом нагрузки между процессами и необходимостью синхронизации. Также существенную роль играют особенности алгоритма: если значительная часть программы остаётся последовательной, общий выигрыш от параллелизации будет ограничен.

**8.В каких случаях использование распределённых вычислений оправдано, а в каких — неэффективно?**

Использование распределённых вычислений оправдано, когда объём данных и вычислений настолько велик, что не помещается или неэффективно обрабатывается на одном узле, например в задачах научного моделирования, обработки больших данных или климатических расчётов. Неэффективным такой подход становится, если задача мала по объёму, требует частых обменов данными или содержит сильные зависимости между шагами, поскольку накладные расходы на коммуникации могут превысить выигрыш от параллельного выполнения.
